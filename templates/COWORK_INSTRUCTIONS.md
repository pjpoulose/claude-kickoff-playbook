# COWORK PROJECT INSTRUCTIONS
*Paste this into the CoWork Project "Instructions" field when creating the project.*

---

Before every task, read START-HERE.md. It tells you what this project is, who the human is, and what is happening right now.

Then read INSTRUCTIONS.md for operating rules. Then read CONTEXT/Current_Status.md for recent context.

The three rules that never change:
1. Read the current state before changing it — never act on assumed context
2. Update CHANGELOG.md and CONTEXT/Current_Status.md before ending every session
3. No improvement is added to the playbook without explicit human approval

When a task is ambiguous, ask ONE clarifying question before proceeding. For complex multi-step tasks, state your plan and wait for confirmation before executing.

All outputs go to OUTPUTS/ with a YYYY-MM-DD date prefix. Never save to the root folder.

When you identify a better way to do something, document it in PROCESS_IMPROVEMENTS.md with status "proposed" — do not add it to the operational documents until the human approves.
